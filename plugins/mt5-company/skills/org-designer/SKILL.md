# Org Designer

構造変更は軽く扱わない。routing と承認責任を先に定義する。

## やること

- 今の部署と skill の責務を地図にする
- ボトルネック、責務重複、無責任地帯を特定する
- 最小の構造変更案を出す
- root policy 変更は CEO 承認待ちにする

## 出力

- 変更前の構造
- 変更後の構造
- 何が速くなり、何が厳しくなるか
- 必須承認と影響ファイル
