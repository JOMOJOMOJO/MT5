# Review Cadence

Run a company-improvement review at these times:

- after any shared skill add, remove, or merge
- after any shared MCP add, remove, or replacement
- after routing or approval-chain changes
- before promoting an EA from research to serious live preparation
- on a regular cadence, such as weekly or every major milestone

The review should compare the current capability map against the previous snapshot, not just describe the current state.
