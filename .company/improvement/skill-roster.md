# Skill Roster Review

## Core Skills To Keep

- `company`
  Entry point and routing anchor. Keep.
- `research-director`
  Owns experiment ladder and search order. Keep.
- `backtest-analysis`
  Owns report reading and result interpretation. Keep.
- `mq5-review`
  Owns MQL quality and execution safety. Keep.
- `risk-manager`
  Owns kill-switches, risk caps, and live failure modes. Keep.
- `release-manager`
  Owns promotion gate and release hygiene. Keep.
- `forward-live-ops`
  Owns demo/live operational discipline. Keep.
- `knowledge-capture`
  Turns raw findings into durable notes. Keep.

## Alpha And Critique Skills To Keep

- `statistical-edge-research`
  Best place for chart-data mining and pre-EA edge discovery. Keep.
- `strategy-critic`
  Necessary because it is allowed to kill weak ideas. Keep.
- `systematic-ea-trader`
  Good for rule discipline and parameter realism. Keep.
- `professional-trader`
  Good for market-fit and execution-realism judgement. Keep.

## Governance Skills To Keep

- `talent-manager`
  Needed when the roster itself changes. Keep.
- `org-designer`
  Needed when workflow or governance changes. Keep.
- `continuous-improvement-office`
  Needed to review the company itself and compare snapshots over time. Keep.

## Do Not Add Yet

- more trader personas with only slight style differences
- multiple near-duplicate optimization or review skills
- a portfolio-level department before there are multiple viable live candidates

These would add ceremony faster than they add decision quality.

## Add Later When Needed

- `portfolio-manager`
  Add only after at least two strategy families are simultaneously close to live or in live operation.
- `broker-operations`
  Add only when real deployment friction, order execution issues, or broker-specific constraints become a recurring blocker.
- `postmortem-facilitator`
  Add only when live/demo incidents become frequent enough that the current review loop is too slow.

## Operating Rule

Before adding a new shared skill, write down:

- what exact decision is not being owned now
- why an existing skill cannot own it cleanly
- what skill should be merged or retired if the new one is accepted
